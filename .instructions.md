# ConfigFiles Repository Guidelines

## Navigation
- This is a **configuration-only repository** — no application code to build or test
- Reference CLAUDE.md for comprehensive repo documentation
- All setup scripts are in `scripts/`;
- All basic configs for home directory, commonly known as dotfiles, are in `configs/`
- Application-specific configs are in `programs/` organized by app (i3, rofi, tilix, etc.)
- Documentation files include main `README.md` and specific `**/README.md` files
  in subdirectories. Also there are general documentation files in `docs/` directory.

## Safe Operations
- Configs should be edited in place and committed — never ask me to deploy directly
- Scripts are NOT idempotent; setup.sh should only run once on fresh systems

## Docker
- Files in `docker/` are still in experimental phase and may not be fully functional or up-to-date, don't rely on them then or read them.

## Archive
- Files in `archive/` are deprecated and should not be used for new setups; they are kept for historical reference and may contain outdated information or configurations.
- Don't read or reference files in `archive/` unless specifically instructed to do so for historical context.

## Reference Files (Read-Only)
- ignore CLAUDE.md - it is specific only for Claude Code, and may not be up to date.
- checklist.md: Post-install manual steps
- security_check.md: Known hardening notes

## When Editing/Writing
- User git email: aj.walkiewicz@gmail.com
- Target OS: Ubuntu/Debian with i3 window manager
- Bash is the primary shell, all scripts should be compatible with bash.
- My favorite shell is ZSH and I use it on my local machine, that's why all
  scripts should use bash command or have shebang `#!/bin/bash` to ensure
  compatibility with my local setup.
